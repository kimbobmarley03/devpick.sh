## Actionable Improvements

### Week 1 Priority (highest impact)
1. **Wait for Google indexing** — sitemap submitted, should start appearing in 3-5 days
2. **Submit to dev tool directories** — awesome lists, Product Hunt, dev.to
3. **Write a launch post** — Reddit r/webdev, r/programming, Hacker News

### SEO Improvements to Implement
1. **Title tag optimization** — Current: "JSON Formatter Online — devpick.sh". Better: "JSON Formatter Online — Format, Validate & Minify JSON Free | devpick.sh"
2. **Internal linking** — Each tool page should link to related tools (e.g., JSON Formatter → YAML to JSON, CSV to JSON)
3. **Content depth** — Add "how it works" sections, examples, and FAQs to each tool page (Google loves comprehensive pages)
4. **Comparison content** — "devpick.sh vs jsonformatter.org" type pages (captures comparison searches)

### LLM Optimization (from the Nate article)
1. **Add a "Why devpick.sh" page** — neutral, factual, Wikipedia-style (brand-facts concept adapted for tools)
2. **Schema markup** — Already have JSON-LD ✅, but could add FAQPage schema
3. **External citations** — Get mentioned on Reddit, Stack Overflow, dev blogs
4. **Machine-readable manifest** — `/.well-known/tools.json` listing all tools with descriptions

### What regex101.com does right (learn from the leader)
- Multi-language support (JS, Python, Go, PHP)
- Code generation ("generate code for this regex")
- Community pattern library
- Detailed explanation of each regex part
- These are features we could add to differentiate
